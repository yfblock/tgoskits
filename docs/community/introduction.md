# 关于我们

TGOSKits 是一个面向操作系统与虚拟化开发的统一工作区。它将 ArceOS、StarryOS、Axvisor 以及大量共享组件 crate 组织到同一仓库中，帮助开发者围绕统一的目录结构、构建入口和验证路径开展协作。

## 我们的愿景

我们相信系统软件研发需要稳定的工程组织、清晰的层次边界和开放的协作方式。通过维护一个开源、统一、可演进的工作区，我们希望：

- **降低系统开发门槛**，让更多开发者能更快理解仓库结构、构建入口和验证方式
- **推动组件化协作**，让共享能力能够在多个系统路径中稳定复用
- **构建开放生态**，围绕内核、虚拟化、平台与测试形成持续协作
- **促进知识共享**，通过文档、教程和实践经验沉淀团队认知

## 社区价值观

我们的社区建立在以下核心价值观之上：

### 开放协作
我们相信开放的力量，鼓励透明的协作和知识共享。无论您是开发者、用户还是爱好者，您的贡献都是宝贵的。

### 技术卓越
我们追求技术卓越，致力于创建高质量、高性能的代码。我们重视代码审查、测试和文档，确保项目的长期可维护性。

### 包容性
我们欢迎来自不同背景、技能水平和经验的社区成员。我们致力于创建一个友好、尊重和包容的环境，让每个人都能做出贡献。

### 持续学习
系统软件工程持续演进，我们鼓励持续学习和知识分享。通过文档、教程和讨论，我们帮助社区成员建立从组件到系统的整体理解。

## 如何参与社区

无论您是开发者、用户还是系统软件爱好者，都可以通过多种方式参与社区：

### 关注项目
- ⭐ 在 GitHub 上关注我们的项目
- 📢 订阅[博客](/blog)获取最新动态
- 📖 阅读[项目概览](/docs/introduction/overview)与[快速开始](/docs/design/reference/quick-start)

### 参与讨论
- 分享您的想法和使用经验
- 提出功能建议和改进意见

- 提交代码修复或新功能（详见[贡献指南](/community/contributing)）
- 编写测试用例，提高代码质量

### 参加活动
- 参与社区组织的技术分享和研讨会
- 关注仓库更新、文档演进和社区讨论
