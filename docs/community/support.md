# 获取帮助

TGOSKits 是一个开源工作区项目，我们希望为用户和贡献者提供清晰、可执行的支持入口。如果您在阅读文档、构建系统、系统运行或组件开发过程中遇到问题，可以通过以下方式获取帮助。

## 支持渠道

### 文档优先
我们的[官方文档](/docs/introduction/overview)提供了详细的使用指南、API 参考和最佳实践。在提问前，请先查阅相关文档，这通常是解决问题最快的方式。

### 问题反馈 (GitHub Issues)
如果您发现了 bug 或有功能建议，请在 [GitHub Issues](https://github.com/rcore-os/tgoskits/issues) 上提交。提交问题时请注意：
- 先搜索是否已有类似问题
- 使用清晰、描述性的标题
- 提供详细的重现步骤、环境信息和错误日志

### 社区讨论
对于一般性问题、想法分享或使用经验交流，欢迎在仓库协作渠道中继续讨论。这里更适合：
- 分享最佳实践和解决方案的平台
- 与其他用户和开发者交流经验的地方
- 提出功能建议和参与技术讨论的空间

如果社区维护了即时交流渠道，也可以在其中继续做问题咨询、经验交流与反馈建议。

## 支持范围

### 社区支持
TGOSKits 主要依靠社区支持，包括：
- 文档和教程
- 社区成员间的互助
- 开源贡献者的技术支持

### 商业支持
如果您有企业级需求或定制开发需求：
- 可以通过 GitHub Issues 或维护者协作渠道描述您的需求
- 考虑赞助项目或雇佣核心贡献者进行定制开发

## 常见问题

### 如何快速开始？
请参考我们的[快速开始指南](/docs/design/reference/quick-start)，其中包含了当前仓库最短的环境准备与运行路径。

### 如何报告安全问题？
如果您发现安全相关的问题，请不要在公开渠道披露。请通过 GitHub 的私有安全报告功能或联系项目维护者。

### 在哪里可以找到更多资源？
查看[社区资源页面](/community/resources)获取工具、教程和第三方集成。

## 获取帮助的最佳实践

1. **先搜索**: 在提问前，先搜索文档、GitHub Issues 和已有讨论
2. **提供详细信息**: 包括您的环境信息、复现步骤和错误日志
3. **使用正确的渠道**: Bug 报告优先使用 Issues，一般问题结合文档和协作渠道继续讨论
4. **保持礼貌**: 尊重社区成员，保持友好和专业的交流方式

我们感谢您对 TGOSKits 的关注和支持！
